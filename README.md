# 每天认3个字

一款专为“不识字或只认识少数字的老年人”设计的手机端生活汉字学习 PWA。无需登录、无需后端，学习记录保存在设备浏览器中。

## 已实现

- 30 天生活课程、90 个汉字学习条目；按汉字本身去重统计真正认识的不同汉字
- 每天 3 字是基础目标；完成后可按组继续学习 3 个新字
- 首页显示今日累计字数，可随时继续学习或复习今天的字
- 每额外学习两组显示休息提醒，家人可设置每日最多 3、6、9 或 15 字
- 启动引导、今日目标、逐字教学、二选一识别、综合复习、今日完成
- 普通话 Web Speech API，慢语速、重复播放、切页停止、无语音时的大喇叭降级
- 教学语音分三段朗读：引导语、目标单字、生活解释；默认停顿 600ms 和 900ms
- 自动评分选择中文自然音色，家人可试听并保存设备上的中文声音
- 本地 MP3 优先、浏览器语音回退，音频加载失败不会阻塞学习
- 答错不批评：重读正确字并允许再次选择
- 每步即时保存；刷新、关闭或中途退出后从原处继续
- 跨日后先复习最多 3 个到期汉字；记录连续学习天数和成长小树
- 新字会自动安排第 1、3、7 天的间隔复习，每天新课前最多复习 3 个到期字，答错字次日再次出现
- 当天答题和长期答题记录分开保存；家人设置按历史错误次数、最近答题日期和正确率显示需要多复习的字
- 连续学习天数只在相邻日期增加，并与累计完成天数分别显示
- 全部新字课程学完后固定进入长期复习首页，不再回到最后一课；长期复习按到期字、易错字和复习间隔组合每日内容
- 总课程指针与每日基础目标分离，额外学习不会在第二天重复出现
- 新字课程与轻量复习课程分开统计；复习会记入练习数，但不会虚增新认识字数
- 长按右上角小房子 3 秒进入家人设置
- 学满 7 天后可切换每天 5 字；支持语速、字体、自动播放、提醒时间设置
- localStorage 损坏或不可用时安全回退，不向老人显示技术错误
- 学习记录使用 version 3；version 1/2 的条目 ID、识字记录和答题记录会自动迁移，同时保留原 ID 用于课程定位和兼容
- 30 秒无操作时语音提醒，最多两次
- 防快速连点、横屏和小屏布局、系统减少动画偏好、无障碍标签
- 可安装 PWA、离线应用外壳、全本地图标与 SVG 场景占位图

## 电脑启动

需要 Node.js 20 或更高版本，以及 pnpm。

```bash
pnpm install
pnpm dev
```

浏览器打开终端显示的网址，通常是 `http://localhost:5173`。

生产构建与预览：

```bash
pnpm build
pnpm preview
```

自动化测试：

```bash
pnpm test
```

## 手机测试

1. 电脑和手机连接同一个 Wi-Fi。
2. 运行 `pnpm dev`。项目已使用 `--host 0.0.0.0` 暴露到局域网。
3. 查看电脑局域网 IP（Windows 可运行 `ipconfig`）。
4. 手机浏览器打开 `http://电脑IP:5173`。
5. 依次测试首次声音、开始按钮、三字学习、答对/答错、刷新恢复和长按家人设置。

语音和 PWA 安装在手机上通常要求 HTTPS；局域网 HTTP 适合页面联调，但完整能力建议部署到带 HTTPS 的静态站点。

## 安装到手机桌面

- Android Chrome：打开 HTTPS 站点，点浏览器菜单中的“安装应用”或“添加到主屏幕”。
- iPhone Safari：点“分享”，选择“添加到主屏幕”。iOS 的语音自动播放限制更严格，首次可能需要先点大喇叭。

应用安装后可离线打开已缓存的程序。浏览器内置中文语音包是否能离线朗读，由手机系统决定。

## 修改课程和汉字

编辑 `src/data/courses.ts`。每个课程包含课程名、今日目标、场景图标、主题和三个汉字。每个汉字数据包含汉字、拼音、解释、生活场景、例句；若省略单独教学语音，程序会自动组合简短朗读文本。

修改后运行：

```bash
pnpm build
pnpm test
```

## 添加本地自然语音 MP3

音频目录已经建立：

```text
public/audio/lessons/      教学引导、解释、例句
public/audio/characters/   单个汉字慢读
public/audio/questions/    题目
public/audio/feedback/     答对与重试提示
public/audio/system/       启动、课程开场和完成语音
```

文件名和完整文案见 `scripts/audio-script.json` 或 `scripts/audio-script.csv`。把生成好的 MP3 按清单的 `saveDirectory` 和 `suggestedFileName` 放入对应目录，然后运行：

```bash
pnpm audio:script
pnpm build
```

第一条命令会重新扫描 MP3、更新 `public/audio/manifest.json`，并把 `exists` 字段改为真实状态。应用只有在清单确认文件存在时才加载 MP3；缺失或播放失败会自动使用 Web Speech API。生产构建会将 MP3 加入 PWA 离线缓存。

优先级为：本地 MP3 → 浏览器中文语音 → 稳定的声音开启遮罩。项目没有云语音 API，也没有任何语音密钥。

## 修改每天学习数量

老人端右上角有一个低干扰的小房子图标，长按 3 秒进入家人设置。每天基础目标固定为 3 字；家人可以把每日最多学习量设为 3、6、9、15 字或不限制。默认不限制，但每额外学习 6 字会显示一次休息提醒。

家人设置还提供“重新体验今天课程”“清除今日学习记录”“跳到下一组课程”和“恢复全部课程”四个测试工具。清除今日记录会保留家人设置；恢复全部课程也会保留当前家人设置。

开发时的默认值在 `src/lib/storage.ts` 的 `defaultSettings` 中。

## 浏览器限制

- Web Speech API 的中文声音、音色和离线能力取决于浏览器与操作系统。
- iOS/部分 Android 浏览器会阻止未经过用户点击的自动语音，应用会显示巨大的喇叭按钮让用户手动触发。
- 网页无法可靠地自行关闭浏览器标签；完成页的“结束”会尝试关闭，浏览器拒绝时保持在明确的完成页面。
- 第一版每日提醒只保存在应用设置中，不做系统级后台推送。
- localStorage 可能在无痕模式或严格隐私设置下被禁用，此时页面仍能学习，但会显示通俗的保存提醒。

## 建议人工真机检查

- Android Chrome 与 iPhone Safari 各一台
- 系统字号放大到 130%–150%
- 320px 宽小屏、横屏、弱网和离线重开
- 有中文语音包和无中文语音包两种设备
- 连续快速点按、系统返回键、中途退出和跨日恢复
